A Step-by-Step Introduction to React Motion (Part I)

2016-04-07
react.js
javascript, react-motion, react.js

react-motion library lets you perform animation in your react apps easily.

Flow Type

react-motion uses flow type, a static type checker by Facebook. You don’t need to use it, but it helps you understand react-motion easier.

react-motion uses flow type to manage its APIs in a strict and brief way.

You can find its Types here.

Let’s take Transition for example.

It tells that Transition’s props should be defaultStyles, styles, children, willEnter, and willLeave.

The question mark (?) means a property is optional; this is, if a property doesn’t have a question mark, it is required.

So, now we know the props of TransitionMotion — you have to fill in styles and its children at least.

Its children should be a function with an array as its argument which renders a react element.

Helpers

spring is the most common used in react-motion.

Used in conjunction with the components below. Specifies the how to animate to the destination value, e.g. spring(10, {stiffness: 120, damping: 17}) means “animate to value 10, with a spring of stiffness 120 and damping 17”.

Its Flow Type tells us: spring should have at least first argument : value (val). And value’s type is number.

It is followed by optional config, a SpringHelperConfig.

SpringHelperConfig has three properties, stiffness, damping and precision (default to 0.01).

react-motion also setups some presets for you. You can use something like spring( x, presets.gentle) ;

Demo

Let’s look at the official demo, Simple Transition. (source code)

It is quite simple. When button is pressed, toggle the state open.

x is with value 400 if this.state.open is true, and on the other hand with value 0 when this.state.open is false.

When this.state.open is changed, the animation starts.

So the square moves from 0 to 400, or 400 to 0 while the button toggles.
