Android Notes: AsyncTask

2015-12-24
Android
AsyncTask, android, java, notes

AsyncTask is a very quick method to do works in a new thread. It is especially useful for accessing network resources.

You can not connect to network if you’re in main thread, which is also called UI thread.

It is “unsafe”.

AsyncTask automatically creates a new thread for you; when your task is done, the thread will be terminated.

Some Part of its Source Code

You can browser Android source code on GitHub.

And here is its Javadoc.

First of all, the AsyncTask class is abstract. So, you cannot create an instance of it and thus you must extend it.

The only method you have to override is doInBackground.

Basic Usage

If you’ve read the source code throughoutly and carefully, you might be as touched as me since Google developers devoted themselves to developing such a great and easy-to-use class.(laugh)

Its basic usage is do whatever you want in the override method doInBackground.

First, when your class extend AsyncTask, you have to decide three data types.

  1. params - the param array when you execute the task.
  2. progress - if you override onProgressUpdate, you will find it really useful.
  3. Result - the doInBackground should return the result when the task is done.
